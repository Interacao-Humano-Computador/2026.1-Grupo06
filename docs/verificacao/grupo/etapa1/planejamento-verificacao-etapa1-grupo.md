# Planejamento da Verificação da Etapa 1 do Grupo

## Introdução

A verificação é uma das etapas mais importantes do desenvolvimento de um projeto. Nela os artefatos produzidos são analisados garantindo que os mesmos cumpram com os seus requisitos especificados. Sendo assim, este documento apresenta o planejamento da verificação dos artefatos produzidos pelo grupo na Etapa 1.

## Objetivos

O objetivo deste documento é apresentar o planejamento para a verificação dos artefatos produzidos na Etapa 1 pelo grupo. Sendo assim, a metodologia, os participantes, os objetos de verificação, o cronograma e as listas de verificação utilizadas na verificação serão mostrados e explicados.

## Metodologia

A metodologia escolhida para esta verificação é a inspeção. Desenvolvida originalmente para códigos de software por Fagan na IBM em 1976, essa técnica consiste em uma revisão formal dos artefatos produzidos a fim de se encontrar defeitos. Essa revisão é realizada através de uma lista de verificação onde se tem uma lista com os defeitos mais comuns que deverão ser identificados, analisados e classificados. Vale ressaltar que a leitura do artefato não deve ser feita pelo autor principal do mesmo.
